# 顺丰快递批量查询系统

基于顺丰速运开放API的快递批量查询前端应用，支持批量查询快递路由信息，提供友好的用户界面和数据导出功能。

## 功能特性

### 核心功能
- ✅ **批量查询**：支持同时查询最多10个快递单号
- ✅ **智能输入**：支持换行、逗号、分号分隔，自动去重和格式验证
- ✅ **实时反馈**：显示查询进度、结果统计和错误提示
- ✅ **路由详情**：展示完整的快递路由信息，包括时间、地点、状态描述
- ✅ **数据导出**：支持导出Excel和PDF格式文件

### 用户体验
- ✅ **响应式设计**：完美适配移动端和桌面端
- ✅ **错误处理**：友好的错误提示和异常处理
- ✅ **频率限制**：防止API滥用，保护服务稳定性
- ✅ **加载状态**：清晰的加载指示和进度反馈
- ✅ **Toast通知**：操作结果的即时反馈

### 技术特性
- ✅ **TypeScript**：完整的类型安全保障
- ✅ **React Hooks**：现代化的状态管理
- ✅ **Tailwind CSS**：美观的界面设计
- ✅ **错误边界**：应用级错误捕获和恢复

## 技术栈

- **前端框架**：React 18 + JavaScript
- **样式框架**：自定义CSS（类Tailwind样式）
- **数据导出**：原生浏览器API（CSV/文本格式）
- **构建工具**：Create React App

## 快速开始

### 环境要求
- Node.js >= 16.0.0
- npm >= 8.0.0

### 安装依赖
```bash
npm install
```

### 真实API集成
当前版本已集成顺丰速运开放API，支持真实的快递查询功能。

**API配置信息**：
- Partner ID: Y5Y030QF
- 已在 `.env` 文件中配置完成
- 支持OAuth2认证和自动Token管理

**使用方法**：
1. 直接启动应用即可使用
2. 输入真实的顺丰快递单号进行查询
3. 查看右上角的API连接状态指示器

### 启动开发服务器
```bash
npm start
```

应用将在 http://localhost:3000 启动。

### 构建生产版本
```bash
npm run build
```

## API配置说明

### 获取顺丰API凭证
1. 访问 [顺丰开放平台](https://open.sf-express.com/)
2. 注册开发者账号并完成认证
3. 创建应用获取 `partnerID` 和 `secret`
4. 申请 `EXP_RECE_SEARCH_ROUTES` 服务权限

### 安全注意事项
- **生产环境**：不要在前端直接暴露API密钥
- **推荐方案**：通过后端代理API调用
- **备选方案**：使用环境变量管理敏感信息

## 使用指南

### 输入快递单号
1. 在输入框中输入快递单号
2. 支持多种分隔方式：
   - 换行分隔
   - 逗号分隔（中英文）
   - 分号分隔（中英文）
3. 系统会自动验证格式并去重

### 支持的单号格式
- 12位纯数字：`123456789012`
- SF开头格式：`SF1234567890`
- 其他常见格式：`AB1234567890`

### 查询结果
- **成功**：显示完整路由信息
- **失败**：显示错误原因和建议
- **部分成功**：显示成功和失败的统计

### 数据导出
- **CSV格式**：包含所有路由详情的表格，可用Excel打开
- **文本格式**：格式化的报告文档
- **文件命名**：自动添加日期标识

## 开发指南

### 添加新功能
1. 在 `types/` 中定义相关类型
2. 在 `components/` 中创建组件
3. 在 `utils/` 中添加工具函数
4. 更新主应用组件集成新功能

### 自定义样式
- 使用 Tailwind CSS 类名
- 自定义样式在 `src/index.css` 中定义
- 响应式设计遵循移动优先原则

### 错误处理
- 使用 ErrorBoundary 捕获组件错误
- 使用 Toast 显示操作反馈
- API错误统一在服务层处理
